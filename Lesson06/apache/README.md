на хост был установлен сервер apache из портов
yum install httpd -y
создан второй файл конфигурации с изменеными настройками на другой порт
При запуске сервера не удалось изменить дефаултовские настройки apache 
Как указано в доке на сайте apache.com 
PidFile=/run/httpd/httpd.pid - не проходит тест

Методом поиска в google пробовал след настройки ничего не помогло.

DEFAULT_PIDLOG=/run/httpd/httpd1.pid
PIDLOG=/run/httpd/httpd1.pid
APACHE_PID_FILE=/run/httpd/httpd1.pid

соответственно при старте сервиса при запущеном сервисе httpd
systemctl start httpd@httpd1.service
не запускается так как процес с пидом уже существует. Если остановить сервис httpd
то сервис запускается на альтернативном порту но вмемте оба сервиса не живут.
Так как оба пытаются писать свой PID в /run/httpd/httpd.pid.
